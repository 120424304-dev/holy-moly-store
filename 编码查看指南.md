# 如何查看和设置文件编码

## Windows 系统

### 查看当前文件编码

**方法1: 记事本查看**
1. 右键 CSV 文件 → `打开方式` → `记事本`
2. 记事本打开后，看右下角状态栏
3. 会显示 `UTF-8` 或 `ANSI`

**方法2: Notepad++ (推荐)**
1. 下载安装 Notepad++ (免费)
2. 用 Notepad++ 打开文件
3. 看右下角，会显示 `UTF-8` 或 `UTF-8-BOM`

**方法3: 命令行查看**
```cmd
# 打开 cmd，进入文件所在目录
chcp 65001  # 设置 UTF-8
file holy-moly-products-usd.csv
```

---

### 转换编码 (如果不对)

**用记事本转换:**
1. 用记事本打开文件
2. `文件` → `另存为`
3. 在底部找到 `编码` 下拉框
4. 选择 `UTF-8`
5. 点击保存，覆盖原文件

**用 Notepad++ 转换:**
1. 打开文件
2. `编码` → `转为 UTF-8 编码`
3. `文件` → `保存`

---

## Mac 系统

### 查看当前文件编码

**方法1: 终端命令**
```bash
# 查看文件编码
file -I holy-moly-products-usd.csv

# 输出示例:
# holy-moly-products-usd.csv: text/plain; charset=utf-8
```

**方法2: VS Code**
1. 用 VS Code 打开文件
2. 看右下角状态栏
3. 会显示 `UTF-8` 或 `GBK` 等

---

### 转换编码

**用 VS Code:**
1. 打开文件
2. 点击右下角编码显示 (比如 "UTF-8")
3. 选择 `通过编码保存` → `UTF-8`

**用终端转换:**
```bash
# 如果文件是 GBK/GB2312，转为 UTF-8
iconv -f GBK -t UTF-8 holy-moly-products-usd.csv > holy-moly-products-usd-utf8.csv
```

## 正确编码的特征

### 用记事本打开应该看到:
- 中文正常显示 (不是乱码)
- 逗号分隔的数据
- 第一行是英文列名

### 用 Excel 打开应该看到:
- 每一列都有对应标题
- 中文显示正常
- 价格显示为数字 ($23.47)

## 快速检查清单

保存文件前确认:
- [ ] 文件名以 `.csv` 结尾
- [ ] 保存类型选择「所有文件」或「CSV」
- [ ] 编码明确选择 `UTF-8`
- [ ] 不要用「文本文档」类型保存

---

## 常见错误

### ❌ 错误: 保存为 `holy-moly-products-usd.csv.txt`
**原因**: 没选「所有文件」，Windows 自动加了 .txt
**解决**: 重新保存，保存类型必须选「所有文件 (*.*)」

### ❌ 错误: 中文变乱码
**原因**: 用了 ANSI 编码
**解决**: 用记事本「另存为」，编码选 UTF-8

### ❌ 错误: Excel 打开 CSV 后中文正常，但 Shopify 导入乱码
**原因**: Excel 自动转换了编码
**解决**: 不要用 Excel 编辑后直接保存，用记事本确认编码是 UTF-8
